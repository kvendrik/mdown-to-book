Analyse

Het probleem is duidelijk, browser testen kost gewoon naast soms veel geld ook veel tijd. Dit hoeft op zich niet een groot probleem te zijn maar omdat het zo veel tijd kost is het fijn om voordat je het testen in gaat al een aantal problemen te kunnen identificeren en uit te kunnen sluiten.

Dit is precies waar Opera Software naar op zoek is, een tool welke kan worden ingezet om ontwikkelaars al voor het testen te informeren over eventuele problemen die zij tegen gaan komen tijdens het testen.

Hierdoor komen er dus tijdens het testen zelf minder onverwachte problemen naar voren en duurt dit proces minder lang wat in sommige gevallen direct geld maar vooral veel tijd kan besparen.

Advies

Mijn advies aan Opera Software is om een tool te maken waar front-end web ontwikkelaars hun JavaScript code in kunnen plakken. Deze word dan gecontroleerd op eventuele punten van verbetering met betrekking tot browser ondersteuning aan de hand van een of meerdere database.

Hierna worden deze verbeterpunten opgelicht en in een lijst verwerkt met uitleg zodat de ontwikkelaar een goed idee krijgt van welke verbeterpunten zijn code bevat en waarom het belangrijk is dat deze verbeterd worden.

Bovenstaand is het alpha product welke aan Opera Software zal worden geleverd en door hun in gebruik zal worden genomen. Deze checkt de gebruikte methodes op caniuse.com en heeft een nauwkeurigheid van zo'n 10%.

Some placeholder (click to zoom)

De nauwkeurigheid van de tool zal in een later stadium worden geoptimaliseerd.Dit doordat er later bijvoorbeeld ook rekening zal worden gehouden met funtionaliteiten die in verschillende browsers anders reageren in plaats van dat er alleen word gekeken naar het ontbreken van gebruikte functionaliteit in de ondersteunde browsers. Bij de mijlpaal van 60% nauwkeurigheid kan de tool open-source worden gereleased.

Een andere uitbreiding die later eventueel zal worden toegevoegt is een grunt module. Deze kunnen ontwikkelaars dan aan hun build script toevoegen zodat het script ook automatisch uitgevoerd kan worden op een aantal megegeven bestanden. Op deze manier kan een bepaalde browser support ook een eis worden voor wanneer bijvoorbeeld een nieuwe versie stabiel kan worden verklaard.
